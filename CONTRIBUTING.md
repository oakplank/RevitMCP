# Contributing

Thanks for improving RevitMCP.

Please open pull requests from a feature branch, not directly from `main`.

In your pull request, include:

*   A short summary of what changed
*   Any relevant Revit versions or setup details
*   Notes about testing or manual verification

Keep changes contained and well described. If a pull request mixes unrelated
fixes, features, formatting, and refactors, it may be harder to review.
